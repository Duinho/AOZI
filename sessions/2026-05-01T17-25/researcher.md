# 📊 최종 데이터 검증 보고서: 운영 효율성 지수(Utilization Index) 및 데이터 무결성 검증

**검증 목표:** Developer가 생성한 3개 보고서(대시보드, Raw Data, 재배치 계획)의 핵심 지표 계산 로직, 데이터 무결성(KPI), 메타데이터 스키마 준수 여부를 검증하여 시스템의 신뢰도를 확보한다.

## 1. 핵심 지표(KPI) 검증: 운영 효율성 지수 (Utilization Index)

| 검증 항목 | 상세 검증 내용 | 검증 결과 | 평가 및 보완 필요 사항 |
| :--- | :--- | :--- | :--- |
| **계산 로직** | $\text{Utilization Index} = \text{실제 수요} / \text{시설 용량}$ | **✅ 적합** | 로직은 정확하게 적용됨. (예: 95/100 = 0.95) |
| **데이터 무결성** | 1. 지수 값이 1.0 초과(과부하)하는 경우의 처리. <br> 2. 데이터 누락 또는 이상치(Outlier) 발생 시의 처리. | **✅ 적합** | 지수가 1.0을 초과하는 항목(과부하)이 명확히 식별되며, 이는 시스템의 핵심 의사결정 근거로 활용 가능함. |
| **보고서 반영** | 3개 보고서 모두 해당 지수를 핵심 메트릭으로 사용하고 있음. | **✅ 적합** | 특히 '재배치 계획' 파일에서 지수가 가장 낮은 (가용 자원) 항목과 가장 높은 (과부하) 항목이 구분되어 활용됨. |

**💡 결론:** 핵심 KPI인 '운영 효율성 지수'는 정의된 공식에 따라 일관성 있게 계산되었으며, 데이터 무결성 측면에서 시스템이 요구하는 수준을 충족함.

## 2. 데이터 구조 및 메타데이터 스키마 검증

| 검증 항목 | 필수 메타데이터 필드 | 포함 여부 | 데이터 표준화 상태 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **필수 필드** | 학년, 성별, 학교, 요일반, 시설\_용량, 실제\_수요, Utilization\_Index | **✅ 모두 포함** | **표준화 완료** | 모든 보고서는 통합 마스터 데이터 스키마를 준수함. |
| **데이터 유형** | 숫자형 (정수/실수), 범주형 (문자열) | **✅ 적합** | **일관성 유지** | 시설 용량 등 수치 데이터가 정수형으로 통일되어 분석에 용이함. |
| **소스 추적성** | 원본 데이터와의 연결고리 | **⚠️ 미흡** | **개선 필요** | 보고서의 각 수치가 어느 원본 파일(예: 엑셀 파일 A의 3번째 행)에서 왔는지 **Source Citation** 메타데이터가 명시적으로 표시되지 않음. |

**💡 결론:** 기본적인 데이터 구조는 표준화되었으나, **'Source Citation'** 메타데이터가 부족하여 보고서의 신뢰성(Trust Score)을 높이기 위한 후속 작업이 필수적입니다.
