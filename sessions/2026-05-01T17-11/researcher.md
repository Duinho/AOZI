**[진단]**
요청된 데이터는 '누가, 언제, 어디서' 이용했는지의 **시간적/공간적 자원 활용 패턴**을 분석해야 의미가 있습니다. 따라서, 단순 엑셀 정리 대신, 모든 데이터가 통합되어 KPI(핵심 성과 지표) 계산에 활용될 수 있는 **통합 데이터 스키마(Master Data Schema)**를 정의하고, 이 스키마를 기준으로 데이터를 구조화하는 것이 최우선 과제입니다.

**[해결책]**
요청하신 데이터를 바탕으로, '시설 용량 대비 실제 수요' 분석을 위한 **통합 데이터 스키마 (Master Data Schema)**와 이를 구축하기 위한 **데이터 전처리 로직**을 제시합니다. 이 스키마가 완성되어야만, 요청하신 형태의 정확하고 신뢰도 높은 결과 보고서(엑셀)가 생성될 수 있습니다.

## 📊 1. 통합 데이터 스키마 정의 (Master Data Schema)

모든 분석의 근거가 되는 표준화된 데이터 구조입니다. 이 스키마는 요청하신 메타데이터와 운영 효율성 분석에 필요한 핵심 지표를 통합합니다.

| 필드명 (Field Name) | 데이터 유형 (Data Type) | 필수 여부 (Required) | 설명 및 포함 정보 (Scope) | 출처 (Source) |
| :--- | :--- | :--- | :--- | :--- |
| **Student_ID** | String (PK) | 필수 | 학생을 식별하는 고유 ID (통합 키) | Raw Data |
| **Generation** | Integer | 필수 | 기수 (1, 2, 3) | Raw Data |
| **Grade** | String | 필수 | 학년 (예: 초1, 중2) | Raw Data |
| **Gender** | String | 필수 | 성별 (남/여) | Raw Data |
| **School_Name** | String | 필수 | 학교명 (예: A중학교) | Raw Data |
| **Day_Type** | String | 필수 | 요일반 분류 (월/화/수/목) | Raw Data |
| **Facility_ID** | String | 필수 | 이용 시설 고유 ID (예: B03_Art) | Raw Data |
| **Facility_Capacity** | Integer | 필수 | 해당 시설의 최대 수용 인원 (용량) | Raw Data |
| **Time_Slot** | String | 필수 | 이용 시간대 (예: 1교시, 오후) | Raw Data |
| **Actual_Demand** | Integer | 필수 | 해당 시간/시설의 실제 이용 인원 (수요) | Raw Data |
| **Utilization_Index** | Float | 계산값 | **(Actual_Demand / Facility_Capacity)**. 운영 효율성 핵심 지표. | **계산 로직** |

## ⚙️ 2. 데이터 전처리 및 구조화 로직 (Processing Logic)

단순히 인원수를 합산하는 것이 아니라, 다음 3단계의 구조화된 프로세스를 거쳐야 합니다.

1.  **ID 통합 및 표준화:** 모든 Raw Data에서 '학생 ID'를 기준으로 학년, 성별, 학교, 요일반 메타데이터를 추출하고 통합합니다. (→ 1차 스키마 완성)
2.  **KPI 지표 매핑:** 시설별 최대 수용 인원(Facility_Capacity) 데이터를 확보하여, 각 학생 이용 기록에 해당 시설의 최대 용량을 매핑합니다.
3.  **지수 계산:** `Utilization_Index = Actual_Demand / Facility_Capacity` 공식을 적용하여, 단순 인원수 대신 **'과부하 위험도'**를 수치화합니다.
