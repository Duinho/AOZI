# 📋 작업 브리프

**원 명령:** 지금 AOZI폴더의 doc파일을 보면 내가 엑셀 몇 개 넣어놨어 이 엑셀을 이해할 수 있어?

## 요약
사용자가 AOZI 폴더에 넣어둔 엑셀 파일들의 데이터 이해 가능 여부를 문의했습니다. AI는 단순한 파일 저장을 넘어, 데이터의 구조화와 목적 정의가 필요함을 설명해야 합니다.

## 분배
- **📱 Secretary**: 사용자 요청을 접수하고, 엑셀 파일을 이해하는 것은 '데이터 로드'를 넘어 '지식 구조화' 과정이 필요함을 안내하십시오. 현재 단계에서는 파일의 종류와 핵심 데이터를 파악하는 것이 최우선 과제임을 강조해야 합니다.
- **🔍 Researcher**: AOZI 폴더의 엑셀 파일들이 어떤 종류의 데이터(예: KPI 결과 보고서, 설문조사 결과, 재무 데이터, 프로젝트 산출물 등)를 담고 있는지 파일 이름이나 샘플 데이터를 기반으로 분류하고, 각 파일에서 추출할 수 있는 핵심 메타데이터 필드(프로젝트명, 기간, 담당 부서, 결과값) 목록을 제안하십시오.
- **💻 Developer**: 기술적인 관점에서 답변해야 합니다. 파일 자체를 '읽는' 것은 가능하지만, AI가 '이해'하고 '사용'하기 위해서는 모든 엑셀 데이터에 대한 '데이터 스키마(Input Schema)' 정의와 '데이터 파이프라인(Data Pipeline)' 구축이 선행되어야 함을 설명하십시오. 사용자에게 파일의 구조(컬럼명, 데이터 타입)를 정의해 달라고 요청하십시오.
- **💰 Business**: 데이터 이해의 궁극적인 목적(Why)을 정의해야 합니다. 사용자에게 '이 엑셀 데이터를 통해 궁극적으로 어떤 의사결정(Decision)을 내리거나, 어떤 질문에 답하고 싶은가?'를 질문하여, 데이터 활용의 비즈니스 목표(KPI)를 명확히 정의하도록 유도하십시오.
