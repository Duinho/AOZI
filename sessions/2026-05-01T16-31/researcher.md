### 1. 📊 데이터 유형 분류 프레임워크 (Classification Framework)

제공된 엑셀 파일들을 구조화하기 위해, 예상되는 데이터 유형을 다음과 같이 분류했습니다. 파일을 이 카테고리에 따라 분류하여 제공해주시면 분석 속도가 극대화됩니다.

| 카테고리 | 데이터 성격 | 포함 예상 내용 | 적합한 메타데이터 추출 예시 |
| :--- | :--- | :--- | :--- |
| **KPI/실적 보고** | 성과 지표 측정 (Quantitative) | 월별/분기별 목표 대비 달성률, 전년 대비 성장률, 핵심 지표 추이. | `KPI 이름`, `측정 단위`, `기준 기간`, `담당 부서`, `달성률` |
| **재무/예산 계획** | 자원 배분 및 회계 (Financial) | 예산 항목별 지출액, 예상 수입, 비용 구조, 재무 성과 요약. | `예산 항목`, `계획 금액`, `실제 지출액`, `분기`, `관련 프로젝트 ID` |
| **프로젝트 산출물** | 업무 프로세스/결과물 (Process/Output) | 기획서 요약, 요구사항 정의서, 테스트 결과, 개발 일정. | `프로젝트명`, `버전`, `담당 PM`, `최종 완료일`, `Scope` |
| **설문/시장 조사** | 사용자/외부 의견 수집 (Qualitative/Survey) | 만족도 점수, 응답자 의견, 시장 트렌드 키워드 빈도 분석. | `설문 주제`, `조사 대상`, `조사 기간`, `평균 점수`, `주요 의견 키워드` |
| **운영 가이드/SOP** | 절차 및 표준화 (Procedure) | 업무 처리 순서, 체크리스트, 내부 정책 문서 요약. | `문서명`, `적용 부서`, `개정 버전`, `효력 발생일`, `주요 업무 흐름` |

---

### 2. 🏷️ 필수 핵심 메타데이터 스키마 정의 (Mandatory Metadata Schema)

모든 파일에서 공통적으로 추출되어야 할 **최소 필수 메타데이터 필드** 목록입니다. 이 필드들이 확보되어야만 파일 간의 관계 설정(Relationship Mapping) 및 통합 분석이 가능합니다.

| 메타데이터 필드 (Field Name) | 데이터 유형 (Data Type) | 필수 여부 | 설명 및 추출 방법 |
| :--- | :--- | :--- | :--- |
| **`Project_ID`** | String (Unique Key) | 필수 | 모든 파일에 부여되는 고유 프로젝트 식별자. (예: AOZI-Q3-2026) |
| **`Project_Name`** | String | 필수 | 프로젝트 또는 보고서의 공식 명칭. |
| **`Reporting_Period`** | Date/String | 필수 | 데이터가 다루는 시간적 범위 (YYYY-MM). |
| **`Department_Owner`** | String | 필수 | 해당 파일의 주 담당 부서 (예: 마케팅, 개발, 재무). |
| **`Data_Type_Category`** | String | 필수 | 위 1번에서 분류한 카테고리 (예: KPI/실적 보고). |
| **`Key_Metric_1`** | Numeric/String | 조건부 | 해당 파일에서 가장 중요한 핵심 지표 1. (예: 매출액, 만족도 점수). |
| **`Key_Metric_2`** | Numeric/String | 조건부 | 해당 파일에서 중요한 보조 지표 2. (예: 성장률, 응답률). |
| **`Source_Link`** | URL | 필수 | 원본 파일의 저장 위치 (Source Citation 확보). |
