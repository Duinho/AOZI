# 📋 작업 브리프

**원 명령:** 자 그럼 1~3기동인 학년별, 성별, 학교별 월수/화목반 분류해서 몇 명씩 있었는지 설명해

## 요약
사용자가 요청한 학년별, 성별, 학교별, 요일반별 학생 인원수를 집계하고 설명하는 보고서를 작성하는 과정입니다. 이 작업은 단순 데이터 추출을 넘어, 내부 데이터 파일의 구조적 분석을 통해 신뢰도를 확보하는 것이 목표입니다.

## 분배
- **🔍 Researcher**: 요청된 학년(1~3학년), 성별, 학교, 요일반(월/수, 화/목)별 학생 인원수를 산출하기 위해, 내부 데이터 파일 목록을 검색하고 해당 정보가 포함된 원본 데이터 파일(예: 엑셀, DB 스키마)을 식별합니다. 데이터의 일관성과 누락된 메타데이터(예: 학년-학교 매핑 테이블)를 확인하여, 데이터 접근 가능성 보고서와 함께 필요한 데이터의 스키마를 정의해 주세요.
- **💰 Business**: Researcher가 제공한 데이터 스키마와 인원수 집계 결과를 바탕으로, 이 보고서가 어떤 비즈니스 질문에 답하는지(KPI)를 정의합니다. 단순히 인원수 보고서에 그치지 않고, '학년별 성비 불균형' 또는 '요일반별 포화도'와 같은 의사결정 시나리오를 제시하고, 해당 지표의 중요도를 분석하여 보고서에 포함할 비즈니스 인사이트를 도출해 주세요.
- **📱 Secretary**: Researcher가 찾은 데이터 소스 정보와 Business가 정의한 KPI 및 분석 포인트를 종합하여, 사용자가 즉시 이해할 수 있는 '공식 보고서' 형식으로 요약하여 작성합니다. 보고서에는 다음과 같은 항목이 반드시 포함되어야 합니다: 1. 최종 집계표 (학년/성별/학교/반별 인원수), 2. 데이터 출처 및 신뢰도 명시, 3. Business가 제시한 핵심 비즈니스 인사이트 (결론 및 제언).
