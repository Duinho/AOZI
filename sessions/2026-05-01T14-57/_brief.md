# 📋 작업 브리프

**원 명령:** 난 내가 다니는 직장의 서류들과 파일들을 전부 숙달하고 질문시 즉각 대답할 수 있을 정도로 학습된 AI를 개발하고 싶어. 예를 들면 결과보고서를 숙달하거나 어떤 사업을 했는지 등을 전부 학습해서
유사 사업에 대해 물어보면 아 비슷한 사업인 이것과 이것을 한 적이 있습니다. 혹은 이 사업의 만족도 결과가   어떻게 나왔지하면 어디 결과보고서에 있는 만족도는 이렇게 나왔습니다 라고 대답했으면 좋겠어.

## 요약
회사의 모든 사내 문서와 데이터를 학습하여, 질문에 즉각적이고 정확하게 답변하는 지식 관리 AI(RAG 시스템) 구축을 목표로 합니다. 핵심은 데이터 수집, 구조화, 그리고 검색 시스템 구축입니다.

## 분배
- **📱 Secretary**: 프로젝트 초기 단계의 업무 정의 및 계획 수립. '사내 지식 기반 AI' 구축을 위한 프로젝트 Scope(범위), 필수 데이터 유형(결과보고서, 사업 계획서, 만족도 설문지 등), 그리고 목표 KPI(검색 정확도, 응답 시간 등)를 정의하고 프로젝트 계획서 초안을 작성할 것.
- **🔍 Researcher**: AI 학습에 필요한 데이터 소스 목록(파일 저장 위치, 형식, 접근 권한 등)을 최대한 상세하게 수집하고, 각 문서 유형별로 어떤 정보(예: 사업명, 기간, 만족도 지표 등)를 추출해야 하는지 데이터 구조화 방안을 리서치하여 보고할 것.
- **✍️ Writer**: 수집된 비정형 데이터(PDF, 스캔된 보고서 등)를 AI가 효과적으로 학습할 수 있도록 표준화된 형식(예: JSON 또는 마크다운 기반의 구조화된 템플릿)으로 변환하는 '데이터 전처리 가이드라인'과 '표준화된 지식 카드 템플릿'을 제작할 것.
- **💻 Developer**: 기술 문서화가 완료된 데이터 전처리 가이드라인과 표준화 템플릿을 기반으로, 사내 문서를 인덱싱하고 질문을 받아 검색 및 답변을 생성하는 RAG(Retrieval-Augmented Generation) 시스템의 프로토타입 파이프라인을 구축하고, API 연결 지점을 설계할 것.
- **✍️ Writer**: 개발된 AI 시스템의 사용자 매뉴얼 및 프롬프트를 작성할 것. 사용자가 '어떤 질문을 어떻게 해야 가장 좋은 답변을 얻을 수 있는지'에 대한 구체적인 가이드와, AI가 전문적이고 신뢰성 있는 톤으로 답변할 수 있도록 시스템 프롬프트(System Prompt)를 설계할 것.
