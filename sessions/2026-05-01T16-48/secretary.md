# 📄 AOZI 운영 자원 효율성 시스템 프로토타입 보고서 (경영진 보고용)

**작성 목적:** 시설/학급 자원의 운영 효율성을 정량적으로 측정하고, 병목 지점(Bottleneck)을 식별하여 자원 증설 및 배정 계획의 근거를 마련함.
**대상:** 경영진 및 프로젝트 관리팀
**작성 일자:** 2026년 5월 1일

## 1. 시스템 작동 원리 및 핵심 성과 지표 (KPI Definition)

본 시스템은 단순히 시설의 최대 용량을 파악하는 것을 넘어, **실제 수요(Demand)** 대비 **가용 용량(Capacity)**의 비율을 분석하여 운영의 과부하 정도를 측정합니다.

**🔑 핵심 지표: 요일반 포화 지수 (Utilization Index)**

$$
\text{Utilization Index} = \frac{\text{실제 배정 인원 (Assigned Student Count)}}{\text{시설 최대 수용 인원 (Max Capacity)}}
$$

| 지수 범위 | 운영 상태 (Operational Status) | 의미 및 경영진 대응 방향 |
| :--- | :--- | :--- |
| $\text{Index} \le 1.0$ | **🟢 안정 (Stable)** | 자원 여유. 현재 배정으로 운영 가능. |
| $1.0 < \text{Index} \le 1.05$ | **🟠 주의 (Caution)** | 용량 근접. 배정 인원 증가 시 모니터링 필요. |
| $\text{Index} > 1.05$ | **🔴 과부하 (Overloaded)** | **긴급 대응 필요.** 시설 용량 초과. 자원 증설(시설/시간) 또는 배정 조정이 필수적임. |

**✅ 시스템 작동 흐름:**
1. **데이터 통합:** 학생 ID를 기준으로, [학생 기본 정보], [시설 용량], [실제 수요] 데이터를 통합합니다.
2. **지수 계산:** 통합된 데이터를 기반으로 `Utilization Index`를 계산합니다.
3. **상태 판정:** 계산된 지수를 위 KPI 기준에 대입하여, 해당 학급/시설의 운영 상태(안정/주의/과부하)를 자동 보고합니다.

## 2. 필수 입력 데이터 목록 및 스키마 (Input Data Schema)

시스템이 정확하게 작동하기 위해서는 다음 세 가지 데이터 소스가 '학생 ID'를 기준으로 표준화되어 통합되어야 합니다.

| 데이터 소스 | 주요 역할 | 필수 필드 (Schema) | 데이터 유형 |
| :--- | :--- | :--- | :--- |
| **학생 기본 정보** | 분석 기준점 및 식별자 제공 | `STUDENT_ID` (Primary Key), `GRADE`, `GENDER` | String, String, String |
| **시설 용량 (Capacity)** | 시설의 물리적 한계 제공 | `FACILITY_ID`, `MAX_CAPACITY` (시설별 최대 수용 인원), `AVAILABLE_DAY` | String, Integer, String |
| **실제 수요 (Demand)** | 현재의 실제 사용량을 제공 | `CLASS_ID` (학급/그룹 ID), `ASSIGNED_STUDENT_COUNT` (실제 배정 인원), `USED_DAY` | String, Integer, String |
